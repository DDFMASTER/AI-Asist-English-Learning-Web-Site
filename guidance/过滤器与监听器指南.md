# AAEL 过滤器与监听器指南

---

## 一、概述

本文档供前端开发人员参考，说明后端 Filter（过滤器）和 Listener（监听器）的实现逻辑，以及新增的管理员监控 API 端点。

自本文档起，后端新增了**服务端会话机制**：用户登录成功后，后端创建 `HttpSession` 并存储用户身份信息。后续请求通过会话 Cookie（`JSESSIONID`）自动携带身份，无需每次请求都传递 `userId` 参数。

> **重要变更：** 登录接口的响应格式不变，但后端会自动在响应头中设置 `Set-Cookie: JSESSIONID=...`。浏览器会自动保存并在后续请求中携带此 Cookie。前端无需额外处理。

## 二、Filter 过滤器

### 2.1 LoginFilter — 登录拦截过滤器

**注解配置：** `@WebFilter("/*")` — 拦截所有请求

**拦截规则：**

| 路径类型 | 是否需要登录 | 说明 |
|---------|------------|------|
| `/api/user/login` | ❌ 公开 | 登录接口 |
| `/api/user/register` | ❌ 公开 | 注册接口 |
| `/api/user/logout` | ❌ 公开 | 登出接口 |
| `/api/connect-test` | ❌ 公开 | 数据库连接测试 |
| `/api/word/*` | ❌ 公开 | 单词搜索/查词 |
| `/api/article/list` | ❌ 公开 | 文章列表 |
| `/api/article/detail` | ❌ 公开 | 文章详情 |
| 其余 `/api/*` | ✅ 需要登录 | 包括评估、个人中心、管理员等 |
| 非 `/api/` 路径 | ❌ 公开 | 静态资源、Vue 页面等 |

**未登录访问受保护 API 的响应（HTTP 401）：**

```json
{
  "success": false,
  "message": "请先登录",
  "code": 401
}
```

**被管理员强制下线后的响应（HTTP 401）：**

```json
{
  "success": false,
  "message": "您已被管理员强制下线，请重新登录",
  "code": 401
}
```

> **前端注意：** 收到 `code: 401` 的响应时，应清除本地用户状态（localStorage 中的 token/userId/role 等）并跳转到登录页。对于"强制下线"消息，可单独弹出提示告知用户原因。

## 三、Listener 监听器

### 3.1 AppContextListener — 全局上下文监听器

**实现接口：** `ServletContextListener`

**触发时机：**

| 事件 | 行为 |
|------|------|
| 应用启动 (`contextInitialized`) | 初始化在线人数计数器、用户会话映射表、强制下线集合、记录启动时间、设置会话超时 30 分钟 |
| 应用关闭 (`contextDestroyed`) | 打印关闭日志，清理所有全局属性 |

**初始化的全局属性：**

| 属性 key | 类型 | 说明 |
|---------|------|------|
| `onlineCount` | `AtomicInteger` | 当前在线会话数 |
| `userSessionMap` | `ConcurrentHashMap<Long, String>` | userId → sessionId 映射 |
| `kickedUserIds` | `ConcurrentHashMap.newKeySet<Long>()` | 被强制下线的用户 ID 集合 |
| `appStartTime` | `LocalDateTime` | 应用启动时间 |

### 3.2 SessionListener — 会话监听器

**实现接口：** `HttpSessionListener` + `HttpSessionAttributeListener`

**功能：**

1. **统计在线人数** — 会话创建时 `onlineCount +1`，会话销毁时 `onlineCount -1`
2. **追踪用户上下线** — 用户登录（设置 `userId` 属性）时建立 `userId → sessionId` 映射
3. **检测重复登录** — 同一用户在新设备登录时，自动清理旧会话映射
4. **超时检测** — 会话销毁时判断是主动登出还是超时离线
5. **清理强制下线标记** — 会话销毁时自动从 `kickedUserIds` 中移除

**会话生命周期：**

```
用户登录 → sessionCreated → attributeAdded("userId")
   │
   ├─ 正常使用（每次请求刷新 lastAccessedTime）
   │
   ├─ 主动登出 → session.invalidate() → attributeRemoved → sessionDestroyed
   │
   ├─ 30分钟无操作 → Tomcat 自动超时 → sessionDestroyed（isTimeout=true）
   │
   └─ 被管理员强制下线 → 加入 kickedUserIds → 下次请求被 LoginFilter 拦截 → sessionDestroyed
```

**会话超时时间：** 30 分钟（在 `AppContextListener` 中通过 `ctx.setSessionTimeout(30)` 设置，`UserLoginServlet` 中也通过 `session.setMaxInactiveInterval(30 * 60)` 设置）。

> **前端注意：** 用户长时间不操作后，会话自动失效。前端可在 AJAX 请求的全局错误拦截中检测 401 状态码，提示用户"登录已过期，请重新登录"并跳转至登录页。

### 3.3 RequestLogListener — 请求日志监听器

**实现接口：** `ServletRequestListener`

**记录内容：**

| 字段 | 说明 |
|------|------|
| `ip` | 客户端真实 IP（优先取 X-Forwarded-For / X-Real-IP，适配 Nginx 反向代理） |
| `path` | 请求路径（如 `/api/user/login`） |
| `method` | HTTP 方法（GET / POST） |
| `timestamp` | 请求时间（精确到秒） |
| `userAgent` | 浏览器 User-Agent（截断至 255 字符） |
| `sessionId` | 会话 ID（未登录时为 null） |
| `username` | 当前登录用户名（未登录时为 null） |
| `queryString` | URL 查询参数 |

**存储方式：** 内存中的同步列表（`Collections.synchronizedList`），最多保留 **1000 条**，超出后自动移除最旧记录。跳过静态资源（.js/.css/.png/.jpg 等）的日志记录，减少无关日志量。

**日志仅供管理员通过 API 查阅，不写入文件。应用重启后日志清空。**

## 四、管理员监控 API

### 4.1 通用约定

所有监控接口均要求 `adminUserId` 参数校验管理员权限。

- **鉴权失败响应：**

```json
{
  "success": false,
  "message": "无管理员权限"
}
```

### 4.2 API 端点详述

#### 4.2.1 查看实时在线用户

```
GET /api/admin/monitor/online-users?adminUserId=<adminId>
```

| 参数 | 必须 | 类型 | 说明 |
|------|------|------|------|
| adminUserId | 是 | Long | 管理员用户 ID |

成功响应：

```json
{
  "success": true,
  "onlineCount": 5,
  "onlineUsers": [
    {
      "userId": 2,
      "username": "zhangsan",
      "role": "normal",
      "studyPurpose": "六级",
      "sessionId": "A1B2C3D4E5F6..."
    },
    {
      "userId": 5,
      "username": "lisi",
      "role": "vip",
      "studyPurpose": "四级",
      "sessionId": "F6E5D4C3B2A1..."
    }
  ]
}
```

**返回字段说明：**

| 字段 | 类型 | 说明 |
|------|------|------|
| onlineCount | Integer | 当前在线会话总数（含未登录的游客会话） |
| onlineUsers | Array | 已登录的在线用户列表 |
| onlineUsers[].userId | Long | 用户唯一 ID |
| onlineUsers[].username | String | 用户名 |
| onlineUsers[].role | String | 角色：normal / vip / admin |
| onlineUsers[].studyPurpose | String | 当前学习阶段 |
| onlineUsers[].sessionId | String | 服务端会话 ID |

> **前端建议：** 在线用户列表用表格展示，role 用彩色 tag（admin=红色, vip=金色, normal=灰色）。可添加"强制下线"操作按钮（调用 4.2.3 接口）。设置自动刷新（建议 30 秒间隔）。

#### 4.2.2 查阅 IP 请求日志

```
GET /api/admin/monitor/request-logs?adminUserId=<adminId>&limit=<limit>
```

| 参数 | 必须 | 类型 | 说明 |
|------|------|------|------|
| adminUserId | 是 | Long | 管理员用户 ID |
| limit | 否 | Integer | 返回条数，默认 100，最大 500 |

成功响应：

```json
{
  "success": true,
  "totalCount": 856,
  "returnedCount": 50,
  "logs": [
    {
      "ip": "192.168.1.100",
      "path": "/AAEL/api/user/login",
      "method": "POST",
      "timestamp": "2026-06-30 18:30:15",
      "username": null,
      "sessionId": null,
      "queryString": null,
      "userAgent": "Mozilla/5.0 (Windows NT 10.0; Win64; x64)..."
    },
    {
      "ip": "192.168.1.100",
      "path": "/AAEL/api/article/list",
      "method": "GET",
      "timestamp": "2026-06-30 18:30:20",
      "username": "zhangsan",
      "sessionId": "A1B2C3D4E5F6...",
      "queryString": "difficulty=四级",
      "userAgent": "Mozilla/5.0 (Windows NT 10.0; Win64; x64)..."
    }
  ]
}
```

**返回字段说明：**

| 字段 | 类型 | 说明 |
|------|------|------|
| totalCount | Integer | 内存中的日志总数（最多 1000） |
| returnedCount | Integer | 本次返回的日志条数 |
| logs | Array | 日志列表（按时间倒序，最新在前） |
| logs[].ip | String | 客户端 IP |
| logs[].path | String | 请求路径（含上下文路径） |
| logs[].method | String | HTTP 方法 |
| logs[].timestamp | String | 请求时间 (yyyy-MM-dd HH:mm:ss) |
| logs[].username | String | 登录用户名（未登录为 null） |
| logs[].sessionId | String | 会话 ID（未登录为 null） |
| logs[].queryString | String | URL 查询参数 |
| logs[].userAgent | String | 浏览器 User-Agent |

> **前端建议：** 表格展示日志列表，支持按 IP、路径、用户名筛选（前端过滤）。可添加"刷新"按钮和自动刷新开关。IP 列可考虑显示为链接样式（但通常不需要实际链接）。

#### 4.2.3 强制用户下线

```
POST /api/admin/monitor/kick-user
```

| 参数 | 必须 | 类型 | 说明 |
|------|------|------|------|
| adminUserId | 是 | Long | 管理员用户 ID |
| targetUserId | 是 | Long | 要强制下线的目标用户 ID |

Content-Type: `application/x-www-form-urlencoded`

成功响应：

```json
{
  "success": true,
  "message": "用户已被强制下线"
}
```

可能失败响应：

| message | 原因 |
|---------|------|
| 缺少目标用户ID | targetUserId 缺失或无效 |
| 不能强制自己下线 | 管理员试图踢自己 |
| 用户不存在 | targetUserId 对应的用户不存在 |
| 不能强制管理员下线 | 目标用户也是管理员 |
| 该用户当前不在线 | 目标用户没有活跃会话 |

**强制下线机制：**

1. 管理员调用此接口 → 目标 userId 加入 `kickedUserIds` 集合
2. 目标用户的会话被从在线映射中移除
3. 目标用户下次发起任何 API 请求时，`LoginFilter` 检测到其已被标记
4. 目标用户的会话被销毁，收到 401 响应："您已被管理员强制下线，请重新登录"
5. 被踢用户的前端检测到 401 → 清除本地状态 → 跳转登录页

> **前端建议：** 在在线用户列表中为每个非管理员用户添加"强制下线"按钮。点击前弹出二次确认对话框："确认强制用户 [username] 下线？该用户下次请求时将自动登出。" 操作成功后刷新在线用户列表。对 admin 角色用户隐藏此按钮。

## 六、前端对接建议

### 6.1 401 全局拦截

建议在 axios 响应拦截器中统一处理 401：

```javascript
// axios 响应拦截器示例
axios.interceptors.response.use(
  response => response,
  error => {
    if (error.response && error.response.status === 401) {
      const data = error.response.data;
      // 清除本地用户状态
      localStorage.removeItem('token');
      localStorage.removeItem('userId');
      localStorage.removeItem('role');
      // 提示用户
      alert(data.message || '登录已过期，请重新登录');
      // 跳转登录页
      router.push('/login');
    }
    return Promise.reject(error);
  }
);
```

### 6.2 管理页面路由建议

```
/admin/monitor          → 监控仪表盘（在线人数概览 + 快捷操作入口）
/admin/monitor/users    → 实时在线用户（表格 + 强制下线按钮 + 自动刷新）
/admin/monitor/logs     → IP 请求日志（表格 + 筛选 + 分页 + 刷新）
```

### 6.3 监控页面设计建议

**在线用户页面 (`/admin/monitor/users`)：**

- 顶部显示在线总人数卡片（大号数字 + "人在线" 标签）
- 用户列表表格：userId、用户名、角色（彩色 tag）、学习阶段
- 每行操作按钮："强制下线"（admin 角色隐藏）
- 自动刷新开关（默认开启，30 秒间隔）
- 手动刷新按钮

**请求日志页面 (`/admin/monitor/logs`)：**

- 顶部显示日志总数
- 筛选栏：IP 输入框、路径输入框、用户名输入框
- 日志表格：时间、IP、方法、路径、用户名
- 分页或"加载更多"按钮（每次获取 100 条）
- 手动刷新按钮

### 6.4 交互规范建议

1. **强制下线确认：** 弹出二次确认对话框，明确告知操作后果
2. **操作反馈：** 成功/失败使用 toast 或 message 组件提示
3. **自动刷新：** 在线用户页面建议 30 秒自动刷新，日志页面手动刷新
4. **错误处理：** 鉴权失败时清除本地状态并跳转登录页
5. **空状态：** 无在线用户或无日志时显示友好的空状态提示

## 七、注意事项

- 请求日志存储在内存中，最多 1000 条，**应用重启后清空**
- 静态资源（.js/.css/.png 等）的请求不会被记录到日志中
- 会话超时时间为 **30 分钟**，超时后用户需重新登录
- 强制下线并非立即生效：目标用户的下一次 API 请求才会被拦截
- 管理员之间不能互相强制下线
- 管理员不能强制自己下线
- 监控接口同样受 `LoginFilter` 保护，必须先登录且有管理员权限
- `onlineCount` 统计的是**所有活跃会话数**（含未登录游客），而 `onlineUsers` 仅列出**已登录用户**
